Get in your script file... or make one first!

First, look down below and be sure your script is actually linked. I can't tell you how many times this has wasted hours of my life trouble shooting something that just wont' work.... only to find out the file isn't being seen by the html.

Step one, let's define some variables!

  • I typically start with the bucket (the element that you are going to be adding the code to). In this case, it is the id "list here"
  • In the script file, type var list = $("#listHere");
  • Why? with jQuery we use the $("") as our "go look for what is in here" # because we are looking for an id, . if class and none if element. And then the name we chose for that "bucket". And finish with the ; always in javascript/jQuery.
  • Second variable ... what is going in here at the highest level? Well, I am making a list which means the list item is the next html needed. So... lets var listItem = $("li /li") but with the alligator brackets in there. (see photo)
  • When do you use those brackets? If the element is in your code and you want to look for it, no brackets. If the element is not in your code and you want to make it... brackets.

Step two, lets use those things!

  • This is where we think about order. First the inside of everything, the smallest nugget. For us, it is the listItem.
  • What do we want to do with it? Give it some formatting? Maybe a class? Let's give it a try.
  • Using the variable name we create we are going to type listItem.text("I am an item"); this code says "the text for this variable should say the words inside the quotation marks"
  • I happen do have a class set up in my css called span1 that adds a background color. We can add that with listItem.addClass("span1"); so inside the li tag, we have added a class of span1
  • Right now... that item is nowhere, we have to put it in our bucket for it to show up. We do this with list.append(listItem); listItem is NOT in "" because it is a variable. If you put it inside "" it will append the words in that string, not the li we created.
  • Couple of notes here... first, everything you want to do to the list item in general needs to be done prior to shoving it in the bucket. Second... had we not made a variable for the bucket you can write the same code by using $("#listHere").append(listItem); and it will do the same thing. When do we make a variable? If you are only doing something once to that bucket... maybe you don't need a variable. But I find it is best to assume you need it again.

So addClass easy enough to understand, you can add whatever you like in there. When do we need to take one out? Most common for me: is-hidden. To toggle things as viewable at certain times you would start with class="is-hidden" and then use your jQuery to .removeClass("is-hidden"); and .addClass("is-visible").

How about .attr() This one is a bit different. It is saying two things at once:
1- $("#listHere").attr("data-id", "hippo"); is going to add the data-id="hippo" to your bucket listHere
2- $("#listHere").attr("data-id"); is going to return the value of the id which is hippo... useful when you are linking multiple buttons into a click event and need specific info about which button was clicked... stuff like that.

.text("") is easy enough, it is going to insert whatever text string is inside

.append("") adds whatever is in there (string, variable, image, etc) to the end of that bucket, it doesn't overwrite current info Ont he flip side .prepend shoves it in front of everything

.empty(); will empty that bucket, generally used prior to the rest of what you want happening there. If you put it at the end of your function, your function will do everything and then empty showing nothing.

.val(); is another multipurpose If you put something in there, it will set a value, if you leave it empty it will return the value.
